# دليل تشغيل ونشر موقع DLM | دلم (النسخة الآمنة مع Firebase)

هذا الدليل يشرح كل خطوة لازمة لتشغيل المشروع فعليًا، من إعداد Firebase حتى النشر على Netlify.

---

## 1) الملفات الموجودة في هذا التسليم

| الملف | الوصف |
|---|---|
| `index.html` | الواجهة الكاملة (نفس التصميم والألوان والحركات الأصلية بدون أي تغيير بصري) |
| `firebase.js` | كل منطق الموقع + Firebase (Auth, Firestore, Storage) في ملف واحد |
| `firestore.rules` | قواعد أمان قاعدة البيانات — يجب نشرها داخل Firebase Console |
| `storage.rules` | قواعد أمان رفع الصور — يجب نشرها داخل Firebase Console |

**لا يوجد أي اسم مستخدم أو كلمة مرور أو "بيانات تجريبية" في أي من هذه الملفات.**

---

## 2) الخطوة الأولى: وضع مفاتيح مشروعك في `firebase.js`

افتح ملف `firebase.js`، ابحث عن أول كتلة فيه باسم `firebaseConfig`، واستبدل القيم بمفاتيح مشروعك الحقيقية (من Firebase Console → ⚙️ Project Settings → عام → Your apps):

```js
const firebaseConfig = {
  apiKey: "PASTE_YOUR_API_KEY_HERE",
  authDomain: "PASTE_YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "PASTE_YOUR_PROJECT_ID",
  storageBucket: "PASTE_YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "PASTE_YOUR_SENDER_ID",
  appId: "PASTE_YOUR_APP_ID"
};
```

> ℹ️ هذه القيم (apiKey, authDomain...) **ليست أسرارًا** بطبيعتها — Firebase صمّمها لتكون مرئية في كود العميل (المتصفح)، والحماية الحقيقية تأتي من **Firestore Security Rules**، لا من إخفاء هذه القيم. لذلك وضعها داخل `firebase.js` آمن تمامًا ومطابق لتوصيات Firebase الرسمية.

---

## 3) تفعيل خدمات Firebase المطلوبة

من Firebase Console، داخل مشروعك:

1. **Authentication** → Sign-in method → فعّل **Email/Password**.
2. **Firestore Database** → Create database → اختر وضع **Production mode** (لا Test mode، لأننا سننشر قواعد أمان مخصصة).
3. **Storage** → Get started (لتفعيل تخزين صور الأغلفة).

---

## 4) إنشاء حساب الأدمن الوحيد (أنت)

هذه هي الطريقة الآمنة الوحيدة لإنشاء حساب أدمن دون كتابة أي بيانات اعتماد في الكود:

1. Firebase Console → **Authentication** → تبويب **Users** → **Add user**.
2. أدخل بريدك الإلكتروني وكلمة مرور قوية (8 أحرف فأكثر، يفضّل أحرف وأرقام ورموز).
3. اضغط **Add user**.

هذا الحساب هو الوحيد القادر على الدخول إلى `/admin`. **لا توجد صفحة "تسجيل" عامة في الموقع عمدًا** — فتح التسجيل للعموم يعني أن أي زائر بالإنترنت يمكنه إنشاء حساب أدمن لنفسه.

لإضافة أدمن ثانٍ مستقبلاً: كرر الخطوات أعلاه بنفس الطريقة من Console.

---

## 5) نشر قواعد الأمان (Firestore + Storage)

### عبر الواجهة (الأسهل، لا يحتاج تثبيت أي أدوات):

1. **Firestore**: Firebase Console → Firestore Database → تبويب **Rules** → امسح المحتوى الحالي → الصق محتوى ملف `firestore.rules` بالكامل → **Publish**.
2. **Storage**: Firebase Console → Storage → تبويب **Rules** → امسح المحتوى الحالي → الصق محتوى ملف `storage.rules` بالكامل → **Publish**.

### عبر سطر الأوامر (اختياري، لمن يفضّل Firebase CLI):

```bash
npm install -g firebase-tools
firebase login
firebase init firestore storage   # اختر المشروع الموجود، واربط الملفين الموجودين بالفعل
firebase deploy --only firestore:rules,storage:rules
```

---

## 6) إنشاء التصنيفات الأولى (مرة واحدة فقط)

الموقع لا يحتوي على أي قصائد أو تصنيفات افتراضية بعد الآن (تم حذف كل البيانات التجريبية من الكود كما طلبت). لإضافة أول تصنيفات:

1. افتح الموقع → اذهب إلى `/?` ثم رابط لوحة التحكم (الزر الموجود في الشريط العلوي، يُظهر "لوحة التحكم").
2. سجّل الدخول بحساب الأدمن الذي أنشأته في الخطوة 4.
3. من تبويب **التصنيفات** داخل لوحة التحكم: أضف تصنيفاتك (مثل: حنين، حب، فراق، وطن، حكمة، أمل) — هذه نفس الأمثلة المستخدمة في النسخة التجريبية الأصلية، يمكنك استخدامها أو تغييرها بالكامل.
4. من تبويب **القصائد**: أضف قصائدك الفعلية، اختر التصنيف، ثم اضغط **نشر القصيدة** (أو **حفظ كمسودة** لو لم تكن جاهزة للنشر بعد).

كل قصيدة تضيفها تظهر فورًا على الموقع للزوار (إن كانت "منشورة")، دون الحاجة لأي تعديل في الكود — هذا هو المطلوب تمامًا.

---

## 7) النشر على Netlify

لا تغيير في طريقة النشر المعتادة:

1. ادفع الملفات الثلاثة (`index.html`, `firebase.js`, ومجلدًا فارغًا إن احتجت) إلى مستودع Git، أو اسحبها مباشرة (Drag & Drop) في واجهة Netlify.
2. **لا حاجة لأي Build Command** — المشروع HTML/JS ثابت بالكامل (Static).
3. Publish directory: المجلد الجذر (الذي يحتوي `index.html`).
4. بعد النشر، أضف نطاق Netlify (مثل `dlm.netlify.app`) أو نطاقك المخصص لاحقًا إلى:
   - Firebase Console → Authentication → Settings → **Authorized domains** → Add domain.
   (هذا ضروري، وإلا سيرفض Firebase طلبات تسجيل الدخول من نطاق غير مُصرَّح به.)

---

## 8) تصحيح تأثير الكتابة العربي — ماذا تغيّر فعليًا؟

**المشكلة الأصلية:** كانت دالة الحركة تُقسّم جملة "لستُ شاعرًا... أنا ترجمانُ شعور." حرفًا بحرف، وتضع كل حرف في عنصر `<span>` منفصل. هذا يكسر "تشكيل الحروف العربي" (Arabic text shaping) لأن المتصفح يحتاج لرؤية الحروف متجاورة فعليًا في نص واحد متصل ليقرر الشكل الصحيح لكل حرف (بداية الكلمة / وسطها / نهايتها)، فتظهر الحروف منفصلة وكأنها مكتوبة بخط القطع المنفصل.

**الحل المطبَّق:** النص الآن عقدة نصية واحدة متصلة 100% (`el.textContent = text`، بلا أي تفكيك). تأثير "الكتابة التدريجية" يتم بالكامل عبر **كشف تدريجي لعرض العنصر** (`clip-path`) بمحاذاة تراعي اتجاه RTL (من اليمين في العربية، من اليسار في الإنجليزية)، مع تسهيل حركة (`ease-out`) عبر `requestAnimationFrame` لحركة سلسة ومتجاوبة على كل الأجهزة. النتيجة: نفس الإحساس البصري الفاخر بالحركة، لكن بدون كسر حرف واحد.

---

## 9) ملاحظات أمان مهمة

- **لا توجد أي بيانات اعتماد في الكود** على الإطلاق — تحقق بنفسك بالبحث عن كلمات مثل `password` أو `admin` داخل `firebase.js`، ستجد فقط أسماء متغيرات ومنطق التحقق، لا قيمًا حقيقية.
- الجلسة تنتهي تلقائيًا بإغلاق المتصفح (`browserSessionPersistence`). إن رغبت ببقاء الجلسة بين الزيارات على جهازك الشخصي، يمكن تغيير هذا السطر في `firebase.js` لاحقًا إلى `browserLocalPersistence` (أقل أمانًا على جهاز مشترك).
- أي محاولة وصول مباشر لرابط `/admin` (عبر `?poem=` أو غيره) من دون تسجيل دخول تُعاد توجيهها فورًا لصفحة الدخول — هذا مطبَّق في طبقتين: الواجهة (`firebase.js`) **و** قواعد Firestore (`firestore.rules`) في حال حاول أي شخص تجاوز الواجهة والوصول للبيانات مباشرة.

---

## 10) جاهزية للمستقبل (كما طلبت)

البنية الحالية لا تحتاج إعادة هيكلة لإضافة:

- **عداد القراءات**: مطبَّق بالفعل (عملية `increment` ذرّية على الخادم).
- **AdSense**: يمكن إدراج كود AdSense مباشرة داخل `index.html` (عادة في `<head>` وأماكن محددة داخل المحتوى) دون أي تعارض مع منطق Firebase.
- **التعليقات / المفضلة / الإشعارات / ملفات تعريف المستخدمين**: تحتاج مجموعات Firestore جديدة بنفس نمط العمل الحالي (`comments`, `favorites`...) + توسيع `firestore.rules` لها، وتفعيل Authentication بطرق إضافية (Google, Anonymous...) إن رغبت بفتحها للزوار لا فقط الأدمن.
- **Analytics**: إضافة `firebase-analytics.js` من نفس عائلة SDK المستخدمة هنا.
- **PWA**: إضافة ملف `manifest.json` و Service Worker لاحقًا، الواجهة الحالية لا تتعارض مع ذلك.
- **نطاق مخصص**: مدعوم بالكامل من Netlify + إضافته كـ Authorized domain في Firebase كما في الخطوة 7.
